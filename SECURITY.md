# Security Policy

## Supported Versions

We release patches for security vulnerabilities. The following versions are currently supported:

| Version | Supported          |
|---------|-------------------|
| 1.x     | :white_check_mark: |

We recommend always using the latest stable release.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### How to Report

1. **Do NOT** create a public GitHub issue for security vulnerabilities
2. Email security concerns privately to the maintainers
3. Include the following in your report:
   - Description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact assessment
   - Any suggested fixes (optional)

### Response Timeline

Our team is committed to responding to security reports:

| Timeline | Action |
|----------|--------|
| **24 hours** | Acknowledge receipt of report |
| **7 days** | Provide initial assessment and timeline |
| **30 days** | Publish security advisory with patch (if applicable) |

### What to Expect

- **Acknowledgment**: You'll receive confirmation that we received your report
- **Status Updates**: We'll keep you informed of our progress
- **Credit**: With your permission, we'll acknowledge your contribution in the security advisory

## Security Best Practices

When using claude-code-Go:

1. **API Keys**: Never commit API keys to version control
2. **Permissions**: Use the minimum required permission level for your use case
3. **Updates**: Keep your installation up to date
4. **Environment**: Review environment variables and configuration files

## Security Contact

For security-related inquiries, please contact the project maintainers through GitHub's private vulnerability reporting or the project's security email.
