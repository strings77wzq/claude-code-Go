# 架构概览

go-code 由多个关键组件组成，它们协同工作，共同实现 Claude Code 的智能体系统。**模型提供智能，harness 提供可靠性**是本项目的核心设计理念。

## 组件说明

### CLI/Repl（命令行界面）

命令行界面是面向用户的入口点，负责：
- 解析命令行参数
- 管理交互式 REPL 会话
- 将用户输入流式传输给智能体

### Agent Loop（智能体循环）

智能体循环是核心编排器，管理智能体的执行周期。它：
- 维护对话历史记录
- 根据模型响应决定调用哪些工具
- 处理工具执行和结果处理
- 管理循环直到任务完成

### API Client（API 客户端）

负责与 Anthropic API 通信：
- 发送消息及工具定义
- 接收流式响应
- 解析模型请求的工具调用
- 处理身份验证和速率限制

### Tool Registry（工具注册表）

所有可用工具的中央注册中心：
- 管理内置工具的注册
- 处理 MCP 工具发现
- 向模型提供工具模式定义
- 将工具调用路由到相应的实现

### Built-in Tools（内置工具）

六个核心工具，提供文件系统和命令执行能力：
- **Read**：读取文件内容
- **Write**：创建或覆盖文件
- **Edit**：进行针对性修改
- **Glob**：按模式查找文件
- **Grep**：搜索文件内容
- **Bash**：执行 shell 命令

### Permission System（权限系统）

确保用户对危险操作的控制：
- 拦截潜在有害的工具调用
- 提示用户确认
- 管理自动批准规则
- 记录权限决策

### Config Loader（配置加载器）

加载和管理配置：
- 读取配置文件（YAML）
- 处理环境变量
- 向所有组件提供配置
- 处理默认值

## 数据流

1. 用户在 CLI/Repl 中输入提示词
2. Agent Loop 接收提示词并添加到历史记录
3. Agent Loop 将完整的消息历史发送给 API 客户端
4. API 客户端向 Anthropic API 发送请求并接收流式响应
5. 如果响应包含工具调用，Agent Loop：
   - 将每个工具调用路由到工具注册表
   - 通过权限系统检查权限
   - 执行工具并捕获结果
   - 将工具结果添加到消息历史
6. 循环继续直到模型提供最终响应
7. 最终响应展示给用户
