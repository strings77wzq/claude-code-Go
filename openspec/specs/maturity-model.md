# Maturity Model & Community Flywheel

Extracted from `open-source-breakthrough-production-roadmap` (archived).

## Five Standout Points

1. Go-native single-binary coding agent
2. Harness-first reliability with Python mock/evaluation infrastructure
3. Real local-agent safety through permission, sandbox, session, and replay
4. China-friendly and global model support (DeepSeek, MiMo, Anthropic, OpenAI-compatible)
5. Bilingual, spec-driven open-source process

## Maturity Levels

| Level | Criteria |
|-------|----------|
| Demo | Runs in a narrow happy path |
| Usable | Quickstart works, core workflows have basic tests |
| Reliable | Harness verifies core trajectories, docs match behavior |
| Production-grade | Release gates cover tests, harness, docs, security, permissions, packaging |
| Community-leading | Benchmarks, examples, contributor flow, roadmap discipline, repeatable releases |

## Wow Path

Users should be able to build, run doctor, execute a mock-provider prompt, inspect a tool call, and replay the session locally — without paid API tokens.

## Community Flywheel

- Clear quickstart docs and demo path
- Benchmark evidence (honest, reproducible)
- Issue/PR templates with good-first-issue labels
- Roadmap clarity and release cadence
- Bilingual contributor onboarding
