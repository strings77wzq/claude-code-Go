## ADDED Requirements

### Requirement: Design philosophy documentation
The website SHALL have a dedicated design philosophy page.

#### Scenario: Core philosophy explained
- **WHEN** a user visits the design philosophy page
- **THEN** they see a deep explanation of "Model provides intelligence, harness provides reliability"

#### Scenario: Five architectural advantages
- **WHEN** a user reads the philosophy page
- **THEN** they see five key advantages: Harness-first engineering, context window management, layered permission defense, local-first execution, MCP as universal bridge

#### Scenario: Design decisions explained
- **WHEN** a user reads the philosophy page
- **THEN** each major design decision is explained with rationale and trade-offs
