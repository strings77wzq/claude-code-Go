## Context

当前官网文档存在 7 个问题需要修复。参考 Claude Code 的五大架构优势，需要让 claude-code-Go 的文档体现出同等深度的架构思考。

## Goals / Non-Goals

**Goals:**
- 架构概览融入五大架构优势（Harness 优先、上下文管理、分层权限、本地优先、MCP 桥接）
- 新增设计理念页面，深度展开"模型提供智能，harness 提供可靠性"
- 合并 MCP + Skills + Hooks 到统一"扩展"目录
- 新增 Skills 技术教程
- 重写核心代码文档（架构思想而非代码翻译）
- 更新 Roadmap（Phase 2 已完成）
- 替换社区页面为反馈渠道

**Non-Goals:**
- 不改 Go 源代码
- 不改官网主题和样式
- 不新增后端功能

## Decisions

### 1. 五大架构优势的表述

**Decision**: 将 Claude Code 的五大优势转化为 claude-code-Go 的表述：

| Claude Code 优势 | claude-code-Go 表述 |
|-----------------|-------------------|
| Harness 优先工程 | 模型提供智能，harness 提供可靠性 |
| 上下文窗口管理 | 智能 Token 估算 + 自动压缩 |
| 分层权限防御 | 三级权限 + glob 规则 + 会话记忆 |
| 本地优先执行 | 纯 Go 单二进制，零运行时依赖 |
| MCP 通用桥接 | MCP 客户端 + 工具自动发现 |

**Rationale**: 借鉴但不抄袭，用我们自己的语言和设计来表述。

### 2. 文档目录结构

**Decision**: 采用"指南→架构→扩展→工具→资源"五层结构。

**Rationale**: 
- 指南：新手入门
- 架构：理解设计思想
- 扩展：自定义和集成（Skills、MCP、Hooks）
- 工具：内置能力
- 资源：路线图和反馈

### 3. 核心代码文档重写策略

**Decision**: 从"代码逐行解释"改为"架构思想阐述"。

**Rationale**: 架构师写文档应该解释"为什么这样设计"，而不是"这行代码做了什么"。代码本身是最好的文档，架构文档应该提供代码无法表达的高层理解。

### 4. 目标读者定位

**Decision**: 从泛泛的"Go 开发者、系统程序员"改为更精准的定位：
- AI Agent 架构研究者
- 对 Harness-First 工程实践感兴趣的资深工程师
- 寻求生产级 AI 工具实现参考的技术负责人

**Rationale**: 更精准，更高大上，符合项目定位。

## Risks / Trade-offs

| Risk | Mitigation |
|------|-----------|
| 文档重构工作量大 | 分优先级，先改最重要的架构概览和设计理念 |
| 设计理念写得太抽象 | 用具体代码示例和设计决策来支撑 |
| 导航变更导致旧链接失效 | 保持旧 URL 的 redirect 或更新所有内部链接 |
