## Context

用户反馈当前官网 features 区块像 README，文字密度太高。参考 5 个顶级 AI 编码工具官网的最佳实践，需要重构为产品级官网。

## Goals / Non-Goals

**Goals:**
- Hero 区域优化（安装命令框 + CTA 按钮）
- 核心数据指标大数字展示
- 功能区块改为独立大区块
- 架构理念区块 + 标签系统
- 终端动画演示
- CTA 底部行动号召

**Non-Goals:**
- 不改 Go 源代码
- 不改文档内容页面
- 不用 React/Vue 重写（纯 VitePress + CSS）

## Decisions

### 1. 功能区块设计

**Decision**: 每个功能独立大区块，底部带标签（aider 风格）。

```
┌──────────────────────────────────────────────┐
│  ⚡ 单二进制部署                               │
│  零依赖，一个文件跑遍全平台。                    │
│  ────────────────────────────────────────── │
│  Linux  macOS  Windows                       │
└──────────────────────────────────────────────┘
```

**Rationale**: 比 6 卡片并排更清晰，每块信息量适中。

### 2. 终端动画

**Decision**: 纯 CSS 动画实现，不依赖 JS 库。

**Rationale**: 性能好，加载快，不需要外部依赖。

### 3. 数据指标

**Decision**: 大字体展示（2.5rem+），4 个指标并排。

**Rationale**: ohmyopenagent 风格的视觉冲击力。

### 4. CTA 底部

**Decision**: 明确的行动号召，3 个按钮（快速开始、查看文档、GitHub）。

**Rationale**: 当前只有版权信息，缺少行动引导。

## Risks / Trade-offs

| Risk | Mitigation |
|------|-----------|
| CSS 动画可能影响性能 | 只用 transform 和 opacity 动画 |
| 暗色主题可能影响可读性 | 保持足够对比度（4.5:1 以上） |
| 终端动画可能太复杂 | 纯 CSS 实现，不依赖 JS |
