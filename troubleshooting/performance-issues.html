Performance Issues ​

Optimizing claude-code-Go performance.

Slow Startup ​

Symptoms ​

  • Takes > 5 seconds to start
  • Shows spinner for long time

Solutions ​

Check binary location:

bash
# Should be native binary, not 'go run'
file $(which go-code)
# Expected: ELF 64-bit executable
1
2
3

Clear old sessions:

bash
rm ~/.go-code/sessions/*.jsonl
1

Disable unused features:

json
{
  "enableMCP": false,
  "enableLSP": false
}
1
2
3
4

High Memory Usage ​

Symptoms ​

  • Memory usage > 500MB
  • System slowing down
  • OOM kills

Solutions ​

Check memory usage:

bash
ps aux | grep go-code
1

Compact context:

> /compact
1

Start fresh:

> /clear
1

Reduce history size:

json
{
  "maxHistoryMessages": 20
}
1
2
3

Slow Responses ​

Symptoms ​

  • Long delays between messages
  • Timeouts

Solutions ​

Check network:

bash
ping api.anthropic.com
1

Increase timeout:

json
{
  "timeout": "60s"
}
1
2
3

Switch model:

> /model claude-haiku-4-6-20251001  # Faster model
1

Simplify requests:

  • Break complex tasks into smaller ones
  • Clear context with /clear
  • Use specific file paths

Token Usage ​

Monitor usage ​

> /tokens

Context: 45,234 / 100,000 tokens (45%)
1
2
3

Reduce usage ​

  1. Compact regularly: /compact
  2. Start fresh: /clear
  3. Be specific: "Read main.go" not "Read all files"

Auto-compact ​

json
{
  "autoCompactThreshold": 0.7  // Compact at 70%
}
1
2
3

Benchmarking ​

Measure performance:

bash
# Startup time
time go-code -p "Hello"

# Memory usage
/usr/bin/time -v go-code -p "Hello"
