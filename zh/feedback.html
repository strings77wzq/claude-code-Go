反馈 ​

有反馈？我们很乐意倾听！

我们使用 GitHub 来管理所有反馈。请根据您的反馈类型使用下面的相应渠道。

问题报告 ​

发现 bug？帮助我们修复它！

渠道： GitHub Issues

使用 GitHub Issues 报告：

  • 崩溃或 panic
  • 意外行为或错误
  • 功能异常
  • 性能问题

提交前：

  • 搜索现有 issues 避免重复
  • 如有可能，包含复现步骤
  • 提及您的 Go 版本和操作系统

功能请求 ​

有新功能的想法？我们很乐意听到！

渠道： GitHub Discussions

使用 GitHub Discussions 建议：

  • 新功能或能力
  • 现有功能的改进
  • 集成想法
  • UI/UX 增强

提交前：

  • 搜索现有讨论避免重复
  • 描述您的功能要解决的问题
  • 考虑它是否符合项目愿景

其他贡献方式 ​

  • ⭐ 在 GitHub 上给项目点 Star
  • 📖 通过 Pull Request 改进文档
  • 🐛 提交 Pull Request 修复 bug
  • 💬 加入讨论并帮助回答问题

感谢您让 claude-code-Go 变得更好！
