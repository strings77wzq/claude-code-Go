贡献者指南 ​

感谢您有兴趣为 claude-code-Go 贡献力量！本指南将帮助您设置开发环境、运行测试并提交拉取请求。

开发环境设置 ​

前提条件 ​

  • Go 1.24 或更高版本 — 安装 Go
  • Git — 版本控制
  • Make — 构建自动化（可选，但推荐）

克隆仓库 ​

bash
git clone https://github.com/strings77wzq/claude-code-Go.git
cd claude-code-Go
1
2

安装依赖 ​

bash
go mod download

构建项目 ​

bash
# 使用 make（推荐）
make build

# 或直接使用 go build
go build -o bin/go-code ./cmd/go-code

代码风格指南 ​

Go 代码规范 ​

  1. 提交前格式化代码：

    bash
    go fmt ./...
    1

  2. 运行静态分析：

    bash
    go vet ./...
    1

  3. 为变量和函数使用有意义的名称

  4. 为导出的函数和类型添加注释：

    go
    // Execute runs the tool with the given input and returns a result.
func (t *Tool) Execute(ctx context.Context, input map[string]any) Result {
    // ...
}
    1
    2
    3
    4

  5. 保持函数专注和简洁

  6. 显式处理错误，不要忽略它们：

    go
    if err != nil {
    return tool.Error(err.Error())
}
    1
    2
    3

提交信息规范 ​

我们遵循 Conventional Commits 以获得清晰且结构化的提交信息。

格式 ​

<type>(<scope>): <description>

[可选的正文]

[可选的页脚]
1
2
3
4
5

类型 ​

类型描述
feat新功能
fix错误修复
docs文档更改
style代码风格（格式化，无逻辑更改）
refactor代码重构
test添加或更新测试
chore维护、依赖项、构建更改

示例 ​

feat(agent): add context compression for long sessions

fix(tool): correct glob pattern matching for hidden files

docs(readme): update installation instructions

refactor(api): simplify SSE streaming parser

chore: update go.mod dependencies

提交拉取请求 ​

提交前 ​

  1. 确保测试通过：

    bash
    go test -v ./...
    1

  2. 运行静态分析：

    bash
    go fmt ./...
go vet ./...
    1
    2

  3. 如果您的更改影响使用，请更新文档

  4. 保持更改专注 — 每个 PR 一个功能或修复

PR 流程 ​

  1. 创建功能分支：

    bash
    git checkout -b feature/your-feature-name
    1

  2. 按照代码指南进行更改

  3. 提交您的更改：

    bash
    git add .
git commit -m "feat(tool): add new tool"
    1
    2

  4. 推送到您的分支：

    bash
    git push origin feature/your-feature-name
    1

  5. 在 GitHub 上打开拉取请求

PR 描述模板 ​

markdown
## 摘要
简要描述此 PR 的内容。

## 相关问题
链接任何相关问题（例如，"Fixes #123"）

## 更改类型
- [ ] 错误修复
- [ ] 新功能
- [ ] 文档更新
- [ ] 重构

## 测试
您如何测试您的更改？

## 检查清单
- [ ] 测试通过
- [ ] 代码已格式化
- [ ] 文档已更新（如适用）
