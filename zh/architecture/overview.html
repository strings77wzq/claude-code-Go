架构概览 ​

在快速演进的人工智能辅助软件工程领域，模型所知道的东西与系统所保证的东西之间的区别，已成为定义架构问题的关键。go-code 秉持一个根本原则：模型提供智能，驾驭系统提供可靠性。

这不仅仅是劳动分工——它是一种架构理念，从权限系统保护危险操作到上下文管理最大化有限上下文窗口的利用率，每个设计决策都遵循这一理念。

go-code 架构的五大支柱 ​

驾驭优先工程 ​

驾驭系统不是事后考虑，而是基础。当构建一个与用户具有相同权限的人工智能代理系统时，可靠性不能层层叠加——它必须从第一行代码开始就嵌入其中。

权限控制确保破坏性操作（文件删除、 shell 命令执行、网络请求）永远无法在没有明确用户同意或适当策略配置的情况下执行。超时保护保证一个行为不当的工具——无论是失控的 grep 搜索还是无限循环——永远不会导致整个会话挂起。输出截断防止单个工具响应消耗整个上下文窗口。会话持久化意味着代理可以在中断后恢复，而不会丢失对话历史或上下文。

这些不是附加在代理循环上的功能。它们是使人 工智能代理能够投入生产的安全网。模型可能会产生幻觉、误解或选择错误的工具——但驾驭系统绝不能在未授权的情况下允许破坏性操作，绝不能无限期挂起，绝不能丢失用户的工作。

上下文窗口作为稀缺资源 ​

现代大语言模型提供慷慨的上下文窗口，但"慷慨"是一个相对术语，当需要代表整个代码库、多个文件和持续的对话历史时。go-code 将上下文视为需要管理的稀缺资源，而不是可以随意挥霍的奢侈品。

该架构实现了延迟工具加载策略——工具注册在中央注册表中，只有工具定义（名称、描述、输入模式）被发送给模型，而不是它们的实现。工具描述本身被限制在 250 个字符以内，强制在与模型沟通功能时保持严谨。

对于长对话，go-code 实现了三级压缩：消息在达到阈值之前保持完整，然后被压缩成在保留关键信息的同时减少 token 计数的摘要。这允许对话持续数十个回合而不会达到 API 限制。

分层权限防御 ​

信任不是二元的，授权也不是。go-code 实施了反映现实安全思维的三级权限模型：

级别功能使用场景
只读文件读取、glob 搜索、grep 搜索安全探索代码库
工作区写入文件创建、修改、编辑受控的开发工作
危险完全访问shell 执行、删除、网络操作需要明确批准的全自动化

权限系统不仅仅检查级别——它根据 glob 规则匹配 评估每个操作，以确定是否允许特定路径或操作。策略可能允许编辑 /src/ 中的文件，同时阻止修改 /config/ 或 /tests/。

会话记忆意味着系统记住会话内的批准决策，防止对同一操作重复提示。存在一种人工在环批准模式，用于需要在执行前明确确认的操作——对于可能危及系统的破坏性操作或 shell 命令至关重要。

本地优先执行 ​

go-code 作为纯 Go 二进制文件运行，零运行时依赖。无需配置 Python 环境，无需维护 Node.js 运行时，无需管理虚拟机。代理在主机完整环境中执行，可以访问所有系统资源、环境变量和已安装的工具。

这种本地优先方法消除了虚拟化开销并提供最大性能。代理可以调用开发者日常使用的相同构建工具、 linter 和测试运行器。模型的意图与系统的实际能力之间没有抽象层。

MCP 作为通用桥 ​

模型上下文协议（MCP）不是一个封闭的生态系统——它是一个互操作性中心。go-code 将 MCP视为外部工具生态系统的桥梁，使代理能够发现和利用其内置功能之外的能力。

可以配置 MCP 服务器来提供专门的工具：数据库访问、API 集成、云服务管理或领域特定的分析器。代理以与内置工具相同的方式对待 MCP 工具——具有完全相同的权限执行保证。

设计理由 ​

为什么选择 Go？ ​

选择 Go 是因为它在性能、简单性和静态链接之间的平衡。代理循环处理实时流式传输、维护对话状态并协调并发工具执行——所有这些都需要可预测的内存使用和低延迟。Go 的单二进制部署模型与本地优先理念一致：无环境配置，无依赖地狱。

为什么是三级权限？ ​

二进制的允许/拒绝模型对于需要在不同上下文中执行各种操作的人工智能代理来说是不够的。开发者可能希望允许在工作区中自动编辑文件，同时要求对 shell 命令进行确认。带有 glob 匹配的分层模型为实用、安全的自动化提供了所需的粒度。

为什么需要上下文压缩？ ​

Token 限制不是理论问题——它们直接影响模型能够推理的内容。如果没有压缩，50 轮的对话将消耗整个上下文窗口，让模型看不到它正在编辑的代码。压缩保留了对话的语义本质，同时为新信息腾出空间。

为什么设置 MAX_TURNS？ ​

模型没有内在的时间或资源理解。如果模型持续选择调用工具，代理循环理论上可以无限期继续。MAX_TURNS 提供了硬边界，防止资源耗尽，同时仍允许复杂任务完成。这是一个独立于模型行为的可靠性保证。
