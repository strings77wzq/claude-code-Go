Agent Loop 核心 ​

Agent Loop 是整个系统的核心执行引擎，负责协调模型、工具和用户之间的交互。

概述 ​

Agent Loop 实现以下循环机制：

实现细节 ​

消息历史（Message History） ​

智能体维护一个消息列表，包含：

  • 用户消息（user）：用户的提示词
  • 助手消息（assistant）：模型的响应
  • 工具结果消息（tool）：工具的输出
go
type Message struct {
    Role    string    // "user", "assistant", "tool"
    Content string    // 文本内容
    ToolID  string    // 工具调用 ID（用于工具结果）
}

工具选择（Tool Selection） ​

当模型响应包含工具调用时，智能体执行以下步骤：

  1. 从模型响应中解析工具调用
  2. 验证工具在注册表中存在
  3. 检查工具操作的权限
  4. 使用提供的参数执行工具
  5. 捕获结果并添加到历史记录

继续条件 ​

循环继续的条件：

  • 模型请求执行工具（tool_use 不为空）
  • 响应不完整（流式传输中）

循环终止的条件：

  • 模型提供最终文本响应（无工具调用，stop_reason 为 end_turn）
  • 发生错误
  • 达到最大迭代次数（防止无限循环）

工具模式（Tool Schema） ​

模型会收到描述所有可用工具的模式定义：

json
{
  "name": "Read",
  "description": "读取文件内容",
  "parameters": {
    "type": "object",
    "properties": {
      "file_path": {
        "type": "string",
        "description": "要读取的文件路径"
      }
    },
    "required": ["file_path"]
  }
}

错误处理 ​

Agent Loop 处理多种错误情况：

错误类型处理方式
API 错误使用指数退避重试
权限拒绝跳过工具执行，通知模型
工具执行失败返回错误给模型，允许恢复
超时返回超时错误给模型

配置项 ​

智能体循环的行为可以通过以下配置进行调整：

yaml
# 每次对话的最大工具调用次数（防止无限循环）
max_iterations: 100

# 工具执行超时时间（秒）
tool_timeout: 30

# 是否显示工具执行进度
verbose: true

stop_reason 详解 ​

模型响应的 stop_reason 字段指示智能体循环的状态：

stop_reason含义后续操作
tool_use模型请求调用工具执行工具并继续循环
end_turn模型提供最终响应循环结束，返回结果
max_tokens达到 token 上限可选择继续或结束
stop_sequence遇到停止序列循环结束
